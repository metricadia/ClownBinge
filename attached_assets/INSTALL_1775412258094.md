## Step 2 — Copy files

Copy every file from this package into your Replit project, preserving the
folder structure. They expect Shadcn UI components at `@/components/ui/*` and
a React Query client at `@/lib/queryClient`.

If your project does not yet have those:

**React Query client** (`client/src/lib/queryClient.ts`):
```typescript
import { QueryClient } from "@tanstack/react-query";

async function fetchApi(url: string) {
  const res = await fetch(url, { credentials: "include" });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}

export const queryClient = new QueryClient({
  defaultOptions: {
    queries: { queryFn: ({ queryKey }) => fetchApi(queryKey[0] as string) },
  },
});
```

**Shadcn UI** — follow https://ui.shadcn.com/docs/installation to get
`Button`, `Input`, `Label`, `Textarea`, `Dialog`, `Avatar`, and `Toast`.

## Step 3 — Register server routes

In your Express server entry file (e.g. `server/index.ts` or `server/routes.ts`):

```typescript
import session from "express-session";
import { registerMetricadiaRoutes } from "./editor-routes";

// Session middleware (add BEFORE routes)
app.use(
  session({
    secret: process.env.SESSION_SECRET || "change-me-in-production",
    resave: false,
    saveUninitialized: false,
    cookie: {
      secure: false,   // set true if using HTTPS in production
      httpOnly: true,
      maxAge: 24 * 60 * 60 * 1000, // 24 hours
    },
  })
);

// Serve uploaded files
app.use("/uploads", express.static("public/uploads"));

// Register all Metricadia editor routes
registerMetricadiaRoutes(app);
```

In `client/src/App.tsx` (using Wouter):

## Step 5 — Connect your database

`editor-routes.ts` has `// TODO` comments wherever DB calls go.
Replace each one with your actual query using your ORM.

**Drizzle ORM example for posts update:**
```typescript
app.put("/api/posts/:id", requireMetricadiaAuth, async (req, res) => {
  const { id } = req.params;
  const { title, content, excerpt } = req.body;
  await db.update(posts).set({ title, content, excerpt }).where(eq(posts.id, id));
  res.json({ success: true });
});
```

**Prisma example:**
```typescript
await prisma.post.update({ where: { id }, data: { title, content, excerpt } });
```

## Step 6 — (Optional) Change the admin password

**Default password:** `KoGAlpha#7`

To change it, set the environment variable `METRICADIA_ADMIN_HASH` to the
SHA-256 hash of `newPassword + TOKEN_SECRET`:

```typescript
// Run this once to get your new hash:
import crypto from "crypto";
const secret = process.env.METRICADIA_TOKEN_SECRET || "metricadia-token-secret-change-me";
const hash = crypto.createHash("sha256").update("YourNewPassword" + secret).digest("hex");
console.log(hash); // paste into METRICADIA_ADMIN_HASH
```

Then in your Replit Secrets panel:
- `METRICADIA_ADMIN_HASH` = the hash output above
- `METRICADIA_TOKEN_SECRET` = a long random string (keep secret)
- `SESSION_SECRET` = another long random string

## PettyID™ Person Profiles

PettyID™ lets you highlight a person's name in an article and attach a pop-up
biographical card (name, photo, description).

**To use it:**
1. In the editor, select a person's name in the text.
2. Click the **PettyID™** button in the toolbar.
3. Upload a photo and add an optional description.
4. Click **Add PettyID™** — the name becomes clickable in published articles.

**To render the pop-up in your article view**, add this to your article page:

```typescript
import { PettyIDPopup } from "@/components/PettyIDPopup";
import { useState } from "react";

// In your article body renderer, after rendering content HTML,
// attach click handlers to span[data-pettyid-name] elements:

useEffect(() => {
  const markers = document.querySelectorAll(".pettyid-marker");
  markers.forEach((el) => {
    el.addEventListener("click", () => {
      const name = el.getAttribute("data-pettyid-name") || "";
      const imageUrl = el.getAttribute("data-pettyid-image") || "";
      const description = el.getAttribute("data-pettyid-desc") || "";
      setPopup({ open: true, name, imageUrl, description });
    });
  });
}, [content]);

// Render the popup:
<PettyIDPopup
  open={popup.open}
  onClose={() => setPopup({ open: false, name: "", imageUrl: "", description: "" })}
  name={popup.name}
  imageUrl={popup.imageUrl}
  description={popup.description}
/>
```

## PettyCode™ Inline Components

PettyCode™ lets you embed React components inside article content via
`[[ComponentName]]` shortcodes.

**To use:**
1. Call `initializePettyCodes()` in your app's entry point.
2. In article body HTML, replace `[[PettyCodeAd]]` tokens before rendering:

```typescript
import { renderPettyCode } from "@/lib/pettycode-registry";

// Split content on [[...]] tokens and render components:
const parts = content.split(/\[\[(\w+)\]\]/g);
const rendered = parts.map((part, i) => {
  if (i % 2 === 1) return renderPettyCode(part, `pettycode-${i}`);
  return <span key={i} dangerouslySetInnerHTML={{ __html: part }} />;
});
```

## Troubleshooting

| Problem | Solution |
|---|---|
| White text on white background | Add `prose-invert` class to the editor wrapper |
| Images not uploading | Check `/uploads` directory exists & Express static middleware is set up |
| Login fails immediately | Confirm `METRICADIA_ADMIN_HASH` env var matches `sha256(password + TOKEN_SECRET)` |
| Session lost on reload (Safari) | The token in sessionStorage is used as fallback — ensure `X-Metricadia-Token` header is sent |
| Editor not appearing | Confirm TipTap packages are installed and imports resolve correctly |
